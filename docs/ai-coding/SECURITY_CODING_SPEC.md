# 安全编码规范

本文件只维护 MCP 服务安全规则。AI 新增或修改接口、工具调用、三方系统 connector、凭据引用、OAuth/token、Webhook、文件、日志、脱敏和测试时，必须先阅读本文件。

## 权限和认证

- 工具目录、工具调用、connector 配置、凭据状态、调用审计和批量同步等接口必须有明确权限码、租户校验和测试覆盖。
- MCP 不实现独立入站鉴权；JWT、三方客户端身份、OpenAPI HMAC 签名校验和 nonce 防重放统一由 `user` 服务负责。
- 外部系统调用 MCP/Kellen 时，必须先通过 `user` 的统一认证链路；服务间需要签名校验时优先复用 `AuthOpenApiRpcService` 或 `/auth/open/signatures/verify`。
- 登录态相关接口必须沿用统一认证上下文；退出登录、用户禁用或密码变更后的 token 清理和失效规则必须与 `user` 服务保持一致。
- MCP 服务不得自行绕过统一认证体系，也不得保留可继续访问工具调用接口的旧 token。
- AI、前端、三方系统回调传入的用户、租户、权限、工具 allowlist、凭据 ID 不可信，必须由后端上下文和受控配置重新判断。

## 工具调用安全

- 工具调用必须校验工具是否启用、调用方是否有权限、租户是否匹配、connector 是否可用、参数是否符合白名单 schema。
- 工具参数不能直接决定服务端访问任意 URL、IP、端口、协议或路径；目标 endpoint 必须来自受控 connector 配置，防止 SSRF。
- 工具调用必须设置超时、最大响应大小、重试上限、熔断和降级策略。
- 批量同步、导出、Webhook 触发和 AI 自动调用必须增加频率限制、数量限制和审计日志，防止资源滥用和业务流程滥用。
- 失败时默认失败关闭；权限、租户、签名、凭据和参数校验异常不得降级为允许调用。

## 出站三方系统凭据

- API key、OAuth access token、refresh token、Webhook secret、签名密钥、数据库密码和 Nacos 密码必须来自 Nacos、环境变量或密钥管理系统，不得写入代码、README 示例真实值或测试数据。
- 不为预留能力新增通用 `mcp-secrets.yaml`；只有出现明确 connector 和明确出站密钥需求时，才按该系统设计配置、加密存储和轮换方案。
- MCP 不保存入站三方客户端 `clientSecret`；入站客户端密钥、签名策略和 nonce 记录归 `user` 管理。
- 凭据引用可以落库保存配置 ID、系统编码、租户、状态、过期时间和脱敏摘要；不得保存可直接调用三方系统的明文密钥，除非已采用项目批准的加密和密钥管理方案。
- OAuth token 刷新必须校验租户、connector、过期时间、刷新结果和失败重试上限，刷新失败不得静默使用过期 token。
- AI 发现已有配置、脚本、SQL、测试或文档中存在疑似密钥、Nacos 地址、数据库连接或生产配置时，只能报告文件和行号并说明风险，不得直接替换为占位符、删除或迁移。

## 数据隔离和字段级授权

- 所有工具目录、connector、凭据索引、调用审计和 token 元数据查询都必须按租户隔离。
- 管理端查询其他租户或跨系统数据必须具备明确管理权限和数据权限，不能只校验已登录。
- 返回 VO、导出记录和审计详情必须执行字段级授权，普通用户不得读取完整凭据摘要、完整三方系统错误、完整请求体、完整响应体、手机号、邮箱、患者信息等敏感属性。
- 查看敏感明文必须设计独立接口或独立动作，不能复用普通详情接口，也不能通过 query 参数控制普通接口返回明文。
- 明文查看接口必须具备专门权限码、登录校验、租户校验、数据权限校验、字段级授权、操作理由、审计日志和必要的二次确认或二次认证。

## 脱敏和日志

- 返回 VO 必须做数据脱敏，不向前端或 AI 返回密码、token、密钥、验证码、完整手机号、完整邮箱、内部地址、异常堆栈等敏感信息。
- 日志、异常、审计记录不得输出密码、token、密钥、验证码、完整手机号、完整邮箱、数据库连接串、完整认证头和内部服务地址。
- 三方系统原始响应可能包含个人信息或敏感业务数据，日志只记录工具名、connector 编码、状态、耗时、错误码和必要追踪字段。
- 数据脱敏必须在 VO 组装、序列化或统一出参层完成，不能只依赖前端隐藏字段。
- 导出、下载、回调回执、审计详情等非页面接口同样必须执行脱敏规则。

## Webhook 和回调

- 回调、Webhook、三方事件和导入数据必须校验签名、时间戳、幂等键和数据完整性，不能只按请求来源 IP 或参数存在判断可信。
- Webhook secret 不得写入仓库或前端配置。
- 回调处理必须具备幂等、防重放和审计记录，避免重复同步或重复触发工具调用。
- 回调异常不得输出完整请求体、完整签名头或三方系统凭据。

## 文件遍历和上传校验

- 文件读取、导入、导出和下载功能必须防止路径遍历；用户输入的文件名只能作为文件名使用，不能直接参与目录拼接，必须校验规范化路径仍在允许目录内。
- 文件上传必须校验大小、扩展名、MIME 类型和实际文件头；禁止上传脚本、可执行文件、HTML、SVG 等可能导致执行或 XSS 的内容，文件名必须重新生成。
- 文件保存目录必须由服务端配置决定，不能由请求参数决定；下载三方同步附件必须校验当前用户或当前租户是否有权访问附件记录。
- 解压、批量导入、模板导入必须防止 Zip Slip、超大文件、空文件、重复文件名覆盖和伪造后缀。
- 上传文件不得直接放在可执行目录或静态公开目录；需要公开访问时必须通过受控下载接口、签名 URL 或对象存储权限策略。
- 解析 Excel、PDF、图片、压缩包等复杂文件时必须限制解析深度、页数、尺寸、行数和内存占用。

## SQL、查询和输入校验

- SQL 查询必须使用 MyBatis-Plus、参数绑定或受控 XML 参数，不得拼接用户输入。
- 分页、排序、关键字、工具名、connector 编码、系统编码、时间范围等查询参数必须限制字段、方向、长度和范围。
- 任何 JSON schema、表达式、规则、模板或脚本能力都必须限制变量、函数、执行时间和可访问类型。
- 禁止反序列化不可信 Java 对象、脚本、表达式、模板或规则。

## 安全配置和供应链

- Spring Security、CORS、Actuator、OpenAPI、错误页、静态资源、上传大小、请求体大小等配置必须按最小暴露原则开启，生产环境不得默认开放调试入口。
- 新增依赖、插件、镜像、脚本和 Gradle/Maven 仓库时必须说明用途，优先使用可信仓库和稳定版本，避免引入未维护、漏洞严重或来源不明的组件。
- 依赖升级必须关注安全公告和破坏性变更，不得为了追最新版本跨大版本引入不兼容风险。
- CI/CD、SQL 脚本、初始化数据和配置变更必须保持可审计、可回滚，不能在构建脚本中明文写入密钥。

## AI Registry 安全

- MCP Server 和 AgentCard 只发布 MCP 工具发现、工具 JSON Schema、协议路径和工具调用入口，不保存三方系统真实凭据。
- MCP Server 默认使用 `mcp-streamable` + Nacos 服务发现 `REF` endpoint；只有无法引用 Nacos 服务发现时才使用 `DIRECT`，且必须确认 `advertise-host` 是服务间可达地址。
- `advertise-host` 必须是服务间可达地址，不能把只适合浏览器或本机调试的地址发布给 AI Registry。
- MCP Server 工具、JSON Schema、协议路径或 endpoint 变化时必须递增 `mcp.server.registry.version`，并验证 Nacos MCP 管理读取得到新版本。
- AgentCard 路径、技能、描述或可达地址变化时必须递增 `card-version` 并验证 `ai` 侧读取得到新卡。
- 发布失败只允许 fail-soft 记录日志，不得因为注册失败而绕过工具权限或直接暴露内部 connector。

## 安全日志、告警和审计

- 登录失败、权限拒绝、跨租户访问、越权访问、token 失效、敏感数据导出、批量同步、工具调用失败、Webhook 签名失败、SQL 注入和 SSRF payload 命中必须记录安全审计日志。
- 安全日志必须包含时间、租户、用户、接口、工具、connector、目标资源、结果、原因和 traceId，但不得包含敏感明文、完整请求体或完整响应体。
- 高风险安全事件必须具备告警或可检索标识，不能只写普通业务日志后无人关注。

## 测试要求

- 新增接口测试时，必须覆盖未登录、无权限、跨租户或替换他人 `id` 的失败场景；只测成功路径不算完成安全验证。
- 工具调用测试必须覆盖无权限调用、跨租户 connector、非法工具名、非法参数 schema、禁用工具和数量限制。
- Connector 测试必须覆盖超时、三方失败码、重试上限、熔断、凭据缺失、凭据过期和脱敏日志。
- Webhook 测试必须覆盖签名错误、时间戳过期、重复事件和篡改 payload。
- 文件上传下载测试必须覆盖非法扩展名、超限大小、伪造 MIME、路径穿越文件名和空文件等场景。
- 查询测试必须覆盖排序字段白名单、SQL 注入 payload、超长关键字和非法时间范围。
- 安全测试还必须覆盖 CSRF、SSRF、限流、批量赋值、字段级越权、异常失败关闭、敏感配置泄露、依赖风险、安全日志审计和 MCP 工具流程滥用。
